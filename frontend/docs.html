Code Quality Metrics

Understanding your code quality metrics helps you write more maintainable software.

📈 Quality Grade (A-F)

Based on cyclomatic complexity and maintainability index:

  • A: Excellent - Low complexity, highly maintainable
  • B: Good - Acceptable complexity, good maintainability
  • C: Fair - Moderate complexity, needs improvement
  • D: Poor - High complexity, difficult to maintain
  • F: Critical - Very high complexity, refactor needed

🔄 Cyclomatic Complexity

Measures the number of independent paths through your code. Lower is better:

  • 1-5: Simple, easy to test
  • 6-10: Moderate complexity
  • 11-20: Complex, consider refactoring
  • 21+: Very complex, high risk

🛠️ Maintainability Index

Composite score (0-100) based on code complexity, lines of code, and Halstead metrics:

  • 80-100: Highly maintainable
  • 60-79: Moderately maintainable
  • 40-59: Difficult to maintain
  • 0-39: Very difficult to maintain

Understanding Error Types

DevNest categorizes feedback into five color-coded types for easy understanding.

🔴
Errors (Critical)
Code will not run. Includes syntax errors, undefined variables, import errors, and attribute errors. Fix these first!
🟠
Warnings (Important)
Code runs but has issues. Includes unused imports, dangerous defaults, redefined builtins, and global variables.
🟢
Optimizations
Make code faster and more Pythonic. Includes list comprehensions, enumerate(), isinstance(), and context managers.
🔵
Conventions (Style)
PEP 8 style violations. Includes naming conventions, missing docstrings, and line length limits.
🟣
Refactors
Code structure improvements. Includes unnecessary else clauses and better context manager usage.

Best Practices

Follow these guidelines to get the most out of DevNest AI and improve your Python skills.

✅ Do's

  • Start with fixing errors first, then warnings
  • Read the educational explanations to understand the "why"
  • Apply optimizations to learn Pythonic patterns
  • Use the history feature to track your progress
  • Test small code snippets to learn specific concepts
  • Compare "before" and "after" code in examples

❌ Don'ts

  • Don't ignore errors to focus on style - fix critical issues first
  • Don't blindly apply suggestions without understanding them
  • Don't optimize prematurely - readable code first, then optimize
  • Don't submit code with sensitive data (passwords, API keys)
  • Don't rely solely on AI - use it as a learning tool
⚠️ Security Reminder

Never paste code containing passwords, API keys, personal data, or sensitive information. Always sanitize your code before analysis.

Frequently Asked Questions

Q: Is my code stored or shared?

Your code is analyzed in real-time and only saved if you explicitly save it to your history. Saved analyses are private to your account and never shared.

Q: What Python versions are supported?

DevNest AI supports Python 3.6+ with best support for Python 3.8-3.11.

Q: How accurate is the AI explanation?

AI explanations are generated by Qwen2.5-Coder, a state-of-the-art code model. While highly accurate, always verify important information against official Python documentation.

Q: Can I use DevNest AI for my homework?

Yes! DevNest AI is designed as an educational tool. It helps you understand and fix your code, not write it for you.

Q: Why do I see duplicate feedback cards?

DevNest intelligently deduplicates similar convention violations (like naming) to show only the first occurrence. All errors and warnings are always shown.

Q: How do I save my analysis?

Your analyses are automatically saved to your account history after logging in. Access them anytime from the History page.
